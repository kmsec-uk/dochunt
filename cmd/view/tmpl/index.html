You can run full text searches on content, title, and links.

Results are shown in reverse chronological order of when they were observed. Pages are 100 items long. These settings are hardcoded and cannot be changed.

Below are some example searches to help you get started:

  • "amex" | Documents containing amex
  • title:"*.docx" | Documents with title ending in .docx
  • links:"https://github.com/*" | Documents containing an external link to GitHub
  • title:interview links:github | Title contains interview and a link contains github

Under the hood, searches are run using SQLITE's full text search (fts5) extension. Refer to SQLITE's documentation for search syntax help.
